Service Order Administration (SOA) Guide

Have you been hearing a lot about SOA and have no idea what it is? Or how it even works? Yes? Read on! And hopefully that changes! IF not… still read on! 😊


Things to Remember:

  • SOE -Service Order Entry
  • OSS -Operational Support System
  • SOA - Service Order Administration
  • IMSI - International Mobile Station Identifier
  • LATA - Local Access Transport Area
  • LRN - Location Routing Number
  • MDN - Mobile Directory Number
  • LNP - Local Number Portability
  • ICP - Inter-carrier Communications Process
  • HLR - Home Location Register
  • INC - Industry Numbering Committee
  • LSR - Local Service Request
  • NPAC - Number Portability Administration Center
  • POS - Point of Sale
  • WPR - Wireless Port Request

Today, we take for granted that we can keep our telephone numbers when we change communications service providers—something that wasn't possible just 15 short years ago. But thanks to the Number Portability Administration Center, the NPAC, the system that enables local number portability, consumers can freely switch between service providers and retain their telephone numbers. The SOA is the interface between each SP’s SOE (Service Order Entry) system and the NPAC database and is the point where a new subscription is sent to the NPAC.

The following transactions occur to support LNP functionality:

SOA requests for subscription administration to the NPAC SMS and responses from the NPAC SMS to the SOA. This includes the capability to: create, cancel, modify and disconnect an SV Create, place SV Create into and out of conflict as well as retrieve and activate SV Create versions.




Whenever any of the other providers contacts the porting team, they will sometimes inquire about port out requests (Freedom->Oher Provider). In those cases, we would check NPAC to see the status of that request. In the example below, we have a request to port out a telephone number, but that request is still pending with no resolution.

Now, we will need to create a response using SOA in order to have this number ported out. Firstly, you will need to access the SOA interface by selecting the “SOA Selection” option highlighted in the screenshot below:

Where it says, “From TN”, you will insert the TN in question and hit “Query” button.

In the screenshot above, after querying we can see the status of the port out request is still pending with a “Request Type” of “Old SO Create”, this means the other provider is waiting for freedom to confirm the port out request. To do this we would select the, “SO Number” and this will take us to a screen where we can create a response for this port out.

In the screenshot above we would select “NPAC Pending” from the dropdown and NOT “NPAC Active”. This will open the section where you would modify the date, in the bottom-right corner of the screenshot below, select the date and time to be “Now” and push the time ahead by “2 mins”, in the example below “15:45” should now be “15”47”. This request to allow the request to process faster, within Two (2 ) minutes.

After adjusting the date and time, make sure you save all changes by hitting “Save”. You should then notice at the top left corner a notification stating that your request has been submitted. IF not then reach out to your TL for assistance.

Once you get the confirmation that the request has been submitted, Query the Telephone Number in ICP to ensure the status of the request is confirmed. See the screenshot below for reference.
